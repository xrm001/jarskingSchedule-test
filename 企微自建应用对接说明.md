# 企微自建应用对接说明

## 1. 整体结构

```text
企业微信工作台
  → 打开老板日程H5（HTTPS）
  → 企业微信OAuth获得当前成员UserID
  → 后端查询角色并签发应用会话
  → 前端调用业务API

业务后端
  → 企业微信API：发送审批、摘要和提醒消息
  → 腾讯云ASR：语音转文字
  → DeepSeek：日程指令结构化解析
  → 数据库与定时任务
```

## 2. 企业微信管理后台配置

1. 在“应用管理 → 自建应用”创建“石总日程”。
2. 可见范围仅选择石总、管理员及管理层部门/标签。
3. 保存 `CorpID`、`AgentID` 和应用 `Secret`；Secret只能保存于服务端。
4. 将正式HTTPS域名配置为应用主页及可信域名。
5. 应用主页指向例如 `https://schedule.company.com/wecom/entry`。
6. 如使用消息按钮或事件回调，配置接收消息服务器URL、Token和EncodingAESKey。

## 3. 登录与角色权限

1. 工作台打开应用时，后端发起企业微信OAuth授权。
2. 回调取得成员 `UserID`，再查询本系统角色表。
3. 角色仅允许：`BOSS`、`ADMIN`、`MANAGEMENT`。
4. 老板审批接口必须再次校验 `UserID` 是否为石总，不能只依赖前端按钮隐藏。
5. 无权限成员返回403页面；离开可见范围不等于可以省略后端鉴权。

## 4. 应用消息

服务端通过企业微信发送应用消息，`touser`填写指定成员UserID：

- 新申请：只发给石总。
- 审批结果：只发给对应申请人。
- 每日09:00摘要：只发给石总。
- 会前60分钟、10分钟：发给石总，可按需要同时发给申请人。
- 勿扰满1小时提醒：只发给石总。

建议消息使用文本卡片或模板卡片，点击跳转到相应H5详情页。消息发送结果和企业微信错误码必须入库，方便追踪漏发。

## 5. 所需环境变量

```env
WECOM_CORP_ID=
WECOM_AGENT_ID=
WECOM_APP_SECRET=
WECOM_CALLBACK_TOKEN=
WECOM_CALLBACK_AES_KEY=
WECOM_REDIRECT_URI=https://schedule.company.com/wecom/callback

TENCENT_SECRET_ID=
TENCENT_SECRET_KEY=
TENCENT_ASR_REGION=ap-guangzhou

DEEPSEEK_API_KEY=
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash

APP_TIMEZONE=Asia/Shanghai
```

## 6. 推荐后端接口

```text
GET  /api/me
GET  /api/boss/schedule?date=2026-07-02
POST /api/boss/status
POST /api/boss/personal-schedules
POST /api/boss/voice-commands
POST /api/boss/voice-commands/:id/confirm
GET  /api/boss/approvals
POST /api/boss/approvals/:id/approve
POST /api/boss/approvals/:id/reject
GET  /api/rooms/availability
POST /api/wecom/callback
```

## 7. 上线顺序

1. 准备域名、HTTPS证书和后端服务器。
2. 完成企微OAuth与角色校验。
3. 接入日程/审批数据库及业务API。
4. 接入企业微信应用消息。
5. 接入腾讯云ASR和DeepSeek。
6. 启用定时提醒任务。
7. 使用测试成员范围灰度验证后再开放给全部管理层。
