# 老板端开发执行清单

## A. 已确认业务规则

- [x] 预约对象仅为石总。
- [x] 只有管理层、石总和管理员能够进入应用。
- [x] 仅石总本人可以审批，管理员不能代审批。
- [x] 拒绝申请不要求填写理由。
- [x] 待审批申请不锁定时段，允许多人申请相同或重叠时间。
- [x] 石总通过一份申请后，所有与该会议时间重叠的待审批申请自动拒绝并通知申请人。
- [x] 原“公开”统一改名为“管理层可见”。
- [x] 老板状态包括：有空、会议中、外出中、勿扰。
- [x] 勿扰不影响预约或审批，仅表示不要当面打扰。
- [x] 勿扰时长可选；未选择时长，满1小时向石总发送状态确认提醒。
- [x] 关闭勿扰后，无进行中行程则变为有空；有行程则按会议/外出状态显示。
- [x] 老板可以自行录入个人行程。
- [x] 语音链路：企微录音 → 腾讯云ASR → DeepSeek V4 Flash → 规则校验 → 老板确认 → 写入日程。
- [x] 每日摘要固定09:00；会前固定60分钟和10分钟提醒。
- [x] 六个会议室作为独立资源管理。

## B. 老板端界面

- [x] 企业微信身份登录页与无权限拦截入口。
- [x] 今日首页：当前状态、今日摘要、日程时间线、待审批数量。
- [x] 状态切换面板：状态、可选持续时间、勿扰说明。
- [x] 个人行程录入：主题、类型、起止时间、可见范围。
- [x] 语音录入入口、识别结果和执行前确认卡片。
- [x] 审批中心：按时段分组展示竞争申请。
- [x] 通过后自动拒绝重叠申请的交互反馈。
- [x] 提醒中心：每日摘要、会前提醒、勿扰超时提醒。
- [x] 设置页：权限说明、会议室资源和企微连接状态。

## C. 后端与数据（真实上线阶段）

- [ ] 企业微信OAuth登录、UserID获取和角色校验。
- [ ] 日程、申请、会议室、提醒、状态和审计日志数据库表。
- [ ] 审批事务：批准一份、锁定石总与会议室、拒绝全部重叠申请。
- [ ] 腾讯云ASR接入。
- [ ] DeepSeek JSON指令解析及服务端Schema校验。
- [ ] 企业微信应用消息发送、按钮回调和发送结果记录。
- [ ] 09:00每日摘要及会前60/10分钟定时任务。
- [ ] 勿扰未设结束时间时的1小时提醒任务。
- [ ] 正式域名、HTTPS、日志、备份和监控。

## D. 验收重点

- [ ] 非授权成员无法打开页面或调用接口。
- [ ] 同一时段允许存在多条待审批申请。
- [ ] 通过一条申请时，不会出现两条重叠的已通过申请。
- [ ] 自动拒绝消息不泄露获批会议的私密信息。
- [ ] 勿扰不阻塞审批、申请及系统提醒。
- [ ] 语音指令未经石总确认不得修改日程。
- [ ] 所有新增、审批、拒绝、状态切换均有审计记录。
